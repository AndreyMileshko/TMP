# RUN DEVELOPMENT

## Команда для Cursor Agent

Выполняй разработку TOP Manufacturing Platform автономно согласно:

- `.cursor/rules/00-tmp-autonomous-governance.mdc`;
- `docs/development-control/STATUS.md`;
- `docs/development-control/WORK-QUEUE.md`;
- `docs/development-control/CONTEXT-MAP.md`;
- Manifest текущего Stage.

## Порядок запуска

1. Проверь готовность контрольных файлов.
2. Определи текущий Stage по `STATUS.md`.
3. Если очередь Stage ещё не декомпозирована, создай задачи по Stage Manifest и утверждённым спецификациям.
4. Выполняй задачи последовательно до одного из условий остановки.
5. После каждой задачи обновляй все контрольные файлы.
6. Не требуй от пользователя подтверждения обычных технических решений.
7. Пользователю задавай только вопросы, которые нельзя разрешить из утверждённой документации.

## Условия остановки

Остановись и выдай пользователю короткий blocker report только если:

- документы противоречат друг другу;
- отсутствует обязательное бизнес-правило;
- требуется изменение архитектуры или публичного API;
- требуется новая неутверждённая зависимость;
- проверка не может быть исправлена в Scope;
- требуется внешнее действие пользователя: пароль, доступ, установка, лицензия или выбор целевой среды.

## Формат blocker report

```text
BLOCKED: <ID>
Task: <TASK-ID>
Reason: <точная причина>
Evidence: <документ/файл/проверка>
Options: <варианты с последствиями>
Recommendation: <рекомендуемый вариант>
Required user decision: <один конкретный вопрос>
```

## Формат промежуточного отчёта

После каждой завершённой задачи не останавливай общий цикл, но обновляй файлы. В чате сообщай кратко:

```text
DONE <TASK-ID>: <результат>
Verification: PASSED
Next: <NEXT-TASK-ID>
```
