# Stage 4 Manifest — Security

## Source

Security Specification, Database Specification, audit rules and Capability contracts.

## Planning domains

1. user domain;
2. role template domain;
3. permission/capability registration;
4. individual grants and revocations;
5. password hashing;
6. authentication session;
7. authorization API;
8. audit integration;
9. persistence and migrations;
10. login/logout UI;
11. user administration UI;
12. security and integration tests.

## Exit criteria

Authentication and centralized permission checks work through public Security API and all changes are audited.
