# UI/UX Specification

**Document ID:** TMP-SPEC-010  
**Status:** Accepted  
**Version:** 1.0

---

# Назначение

Документ определяет архитектуру пользовательского интерфейса TOP Manufacturing Platform (TMP).

Все функциональные модули обязаны соответствовать данным правилам.

---

# Основные принципы

1. TMP является Desktop-приложением.
2. Используется JavaFX 21.
3. Все экраны создаются через FXML.
4. Оформление выполняется только средствами CSS.
5. Навигация централизована.
6. UI не содержит бизнес-логики.
7. Интерфейс должен одинаково работать для всех Capability.

---

# Архитектура UI

Используется следующая схема:

```text
FXML
    ↓
Controller
    ↓
ViewModel
    ↓
Application Service
    ↓
Domain
```

# Навигация

Навигация строится автоматически на основании зарегистрированных Capability.

Каждый модуль предоставляет одну точку входа.

Одновременно открыт один рабочий экран.

После повторного запуска приложения открывается последний использованный модуль.

Недоступные пользователю Capability не отображаются.

---

# Экраны

Каждый экран состоит из:

```text
Screen.fxml
ScreenController.java
ScreenViewModel.java
```

Каждый экран имеет:

- один FXML;
- один Controller;
- одну ViewModel.

---

# FXML

Все FXML загружаются только через Navigation Service.

Прямая загрузка FXMLLoader в прикладном коде запрещена.

Повторно используемые элементы интерфейса оформляются отдельными FXML-компонентами.

---

# Controller

Controller отвечает только за:

- обработку событий;
- привязки элементов;
- взаимодействие с ViewModel.

Controller запрещено:

- выполнять бизнес-логику;
- обращаться к Repository;
- обращаться к базе данных;
- использовать Spring Context;
- использовать ApplicationContext;
- использовать BeanFactory.

---

# ViewModel

ViewModel:

- взаимодействует только с Application Service;
- не зависит от JavaFX-компонентов;
- не содержит бизнес-логики Domain;
- легко тестируется отдельно от интерфейса.

---

# JavaFX и Spring

Spring используется только для создания Application Services и ViewModel.

JavaFX Controller не является Spring Bean.

Интеграция JavaFX и Spring выполняется через отдельный Bootstrap.

---

# Обязательные технические экраны

На этапе Development Foundation должны существовать:

- экран входа;
- главное окно;
- экран состояния платформы;
- экран технического журнала;
- экран отсутствия доступа;
- экран "Модуль не реализован";
- экран критической ошибки запуска.

---

# Сообщения пользователю

Поддерживаются четыре категории сообщений:

- информация;
- предупреждение;
- ошибка;
- критическая ошибка.

Правила:

- ошибки валидации отображаются рядом с полем;
- бизнес-ошибки отображаются пользователю понятным языком;
- технические ошибки не содержат Stack Trace;
- успешные операции отображаются неблокирующим уведомлением;
- чувствительные данные никогда не отображаются.

---

# Длительные операции

Во время выполнения длительных операций:

- интерфейс остаётся отзывчивым;
- отображается индикатор выполнения;
- допускается отмена операции, если это предусмотрено бизнес-логикой.

---

# Общие требования

Запрещено:

- создавать бизнес-логику в UI;
- обращаться к Domain напрямую;
- обращаться к Infrastructure напрямую;
- загружать экраны вне Navigation Service;
- создавать статическое меню модулей.

Каждый новый модуль должен автоматически интегрироваться в навигацию после регистрации Capability.

---

# Architecture Rules

**AR-001**

UI взаимодействует с системой только через Application Services.

**AR-002**

Controller не содержит бизнес-логики.

**AR-003**

Все экраны создаются через FXML.

**AR-004**

Все FXML загружаются через Navigation Service.

**AR-005**

Навигация строится автоматически на основании зарегистрированных Capability.
