**Status:** Active

# Назначение

Данный документ является главным чек-листом реализации TOP Manufacturing Platform.

Он используется на протяжении всей разработки и является единственным источником информации о текущем состоянии реализации проекта.

Каждый пункт может быть отмечен выполненным только после полного соответствия критериям завершения (Definition of Done).

---

# Правила работы

1. Реализация ведётся строго последовательно.
2. Новый этап начинается только после завершения предыдущего.
3. Cursor AI работает только в рамках текущего этапа.
4. Запрещено выполнять задачи будущих этапов.
5. После завершения каждой задачи обязательно обновляются:
  - данный чек-лист;
  - журнал реализации;
  - техническая документация (при необходимости).
6. Любое изменение архитектуры требует обновления соответствующей спецификации до продолжения разработки.

# Этап 9. Analytics

## Подготовка

- Реализация выполняется по `Analytics-Specification.md`.
- Проверено завершение Stage 6 — Warehouse.
- Проверено наличие необходимых read-only Public API Warehouse.
- Scope этапа ограничен отчётом «Движение материалов».
- Открытые архитектурные вопросы отсутствуют.

---

## Модуль Analytics

- Создан модуль Analytics.
- Модуль подключён к Maven Reactor.
- Соблюдены разрешённые направления зависимостей.
- Analytics зарегистрирован как Capability.
- Экран Analytics зарегистрирован в навигации приложения.
- Analytics не содержит функций изменения доменных данных.

---

## Права доступа

- Зарегистрировано право `analytics.material-movements.view`.
- Зарегистрировано право `analytics.material-movements.export`.
- Раздел Analytics скрыт без права просмотра.
- Экспорт недоступен без права экспорта.
- Рекомендуемым ролям назначены права просмотра и экспорта.
- Ограничение доступа по отдельным складам не реализовано.

---

## Интеграция с Warehouse

- Движения материалов получаются через Public API Warehouse.
- Текущие остатки получаются через Public API Warehouse.
- Прямой доступ к таблицам Warehouse отсутствует.
- Прямой доступ к репозиториям Warehouse отсутствует.
- Analytics не изменяет складские документы.
- Analytics не изменяет Warehouse Movement.
- Analytics не изменяет Stock Position.
- В отчёт включаются только действующие проведённые движения.
- Корректирующие и обратные движения отображаются отдельными строками.

---

## Категории движений

- Реализована категория «Приход».
- Реализована категория «Расход».
- Реализована категория «Перемещение».
- Реализована категория «Изменение состояния».
- Сохраняется исходный тип Warehouse Operation.
- Внутреннее перемещение не изменяет общий остаток компании.
- Резервирование не изменяет физический остаток.
- Снятие резерва не изменяет физический остаток.

---

## Фильтры отчёта

- Реализован обязательный фильтр по периоду.
- По умолчанию установлен текущий месяц.
- Реализован фильтр по складу.
- Реализован фильтр по материалу.
- Реализован фильтр по категории движения.
- Реализован фильтр по Customer Order.
- Реализован фильтр по Order Item.
- Реализовано применение фильтров.
- Реализован сброс фильтров.
- Реализовано повторное обновление данных.

---

## Сводные показатели

- Рассчитывается остаток на начало периода.
- Рассчитывается приход за период.
- Рассчитывается расход за период.
- Рассчитывается внутреннее перемещение.
- Рассчитывается остаток на конец периода.
- Рассчитывается количество операций.
- Итоги рассчитываются по всей выборке.
- Разные единицы измерения не суммируются.
- Количества рассчитываются с использованием `BigDecimal`.
- Точность исходных складских данных не изменяется.

---

## Таблица движений

- Реализована таблица движений материалов.
- Отображаются дата и время движения.
- Отображаются категория и исходный тип операции.
- Отображаются код и наименование материала.
- Отображается цвет материала.
- Отображаются количество и единица измерения.
- Отображаются склад-источник и склад-получатель.
- Отображается документ-основание.
- Отображаются Customer Order и Order Item при наличии связи.
- Отображается пользователь, выполнивший операцию.
- По умолчанию применяется сортировка от новых движений к старым.
- Реализована серверная сортировка.
- Реализована серверная пагинация.
- Размер страницы по умолчанию составляет 100 записей.
- Поддерживаются размеры страницы 50, 100 и 200 записей.
- Полная выборка не загружается в память для отображения таблицы.

---

## Детали движения

- Реализована read-only карточка движения.
- Отображаются склад и ячейка.
- Отображаются материал и партия.
- Отображаются количество и единица измерения.
- Отображается документ-основание.
- Отображаются связанные Customer Order и Order Item.
- Отображаются пользователь и технические идентификаторы.
- Переходы к связанным объектам учитывают права соответствующих Capability.
- Редактирование движения из Analytics невозможно.

---

## Экспорт XLSX

- Реализован экспорт отчёта в XLSX.
- Экспорт использует применённые фильтры.
- В файл включается полная выборка, а не только текущая страница.
- В файл включаются сводные показатели.
- В файл включаются применённые фильтры.
- В файл включается дата формирования.
- Технические идентификаторы не экспортируются.
- Количества экспортируются без потери точности.
- Строки XLSX записываются последовательно без загрузки всей выборки в память.
- При отсутствии данных файл не создаётся.
- PDF и CSV не реализованы.

---

## Пользовательский интерфейс

- Реализован экран `Аналитика → Движение материалов`.
- Реализован индикатор загрузки.
- Загрузка не блокирует JavaFX Application Thread.
- Реализовано состояние успешной загрузки.
- Реализовано состояние пустого результата.
- Реализовано состояние ошибки получения данных.
- Реализовано состояние ошибки формирования XLSX.
- Новый запрос не может быть перезаписан результатом более старого запроса.
- Редактирование данных через экран отсутствует.

---

## Хранение и аудит

- Analytics не создаёт собственное постоянное хранилище движений.
- Analytics не создаёт собственное постоянное хранилище остатков.
- Analytics не создаёт собственные таблицы экспортов.
- Analytics не создаёт собственные таблицы сохранённых фильтров.
- Отдельные миграции аналитического хранилища отсутствуют.
- Открытие отчёта не фиксируется.
- Изменение фильтров не фиксируется.
- Экспорт отчёта не фиксируется.
- Analytics не дублирует аудит Warehouse и Document Engine.

## Тестирование

- Реализованы unit-тесты расчёта сводных показателей.
- Проверен расчёт остатков на начало и конец периода.
- Проверено межскладское перемещение.
- Проверено резервирование и снятие резерва.
- Проверены разные единицы измерения.
- Проверены все фильтры.
- Проверена сортировка.
- Проверена пагинация.
- Проверены права просмотра и экспорта.
- Проверено отсутствие доступа без соответствующих прав.
- Проверен экспорт полной выборки в XLSX.
- Проверено отсутствие файла при пустом результате.
- Выполнены интеграционные тесты с Warehouse.
- Проверено появление нового движения после обновления отчёта.
- Проверено, что Analytics не изменяет данные Warehouse.
