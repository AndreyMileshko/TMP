# TMP-003 Glossary

**Document ID:** TMP-003
**Status:** Accepted
**Version:** 1.3

## TOP Manufacturing Platform (TMP)
Единая производственная платформа.

## Platform Core
Инфраструктурное ядро платформы без предметной бизнес-логики.

## Platform Component
Компонент, регистрируемый Platform Core через единый контракт.

## Platform Services
Общие сервисы платформы.

## Functional Area
Предметная функциональная область (Warehouse, Production, Procurement и др.).

## Capability
Законченная функциональная возможность платформы, объединяющая команды, представления, документы, права, настройки, отчеты и обработчики.

## Capability Engine
Компонент регистрации и управления Capability.

## Security
Компонент управления пользователями, ролями и правами.

## Workspace
Рабочее место пользователя, формируемое из Capability согласно правам.

## Business Object
Долговременная бизнес-сущность платформы.

## Документ
Объект платформы, изменяющий бизнес-состояние предприятия.

## Document Engine
Компонент управления жизненным циклом документов.

## Document Processor
Единственный обработчик бизнес-логики конкретного типа документа.

## Проведение документа
Изменение бизнес-состояния через Document Processor.

## Order Management
Функциональная область управления заказами.

## Заказ клиента
Контейнер коммерческой информации и позиций заказа.

## Позиция заказа
Главный производственный объект платформы.

## Спецификация
Описание состава изделия.

## Материал
Номенклатурная единица.

## Основной склад
Склад хранения материалов.

## Склад производства
Склад, с которого списываются материалы на производство.

## Карта раскроя
Необязательный документ с рекомендуемым раскроем.

## Выпуск изделий
Документ завершения производства и списания материалов.

## Пользователь
Субъект, работающий в системе.

## Роль
Шаблон набора прав.

## Право
Уникальное разрешение, объявляемое Capability.

## Индивидуальные права
Фактический набор прав конкретного пользователя.


## Warehouse
Функциональная область платформы, являющаяся единственным владельцем складских остатков.

## Storage Cell
Ячейка хранения материалов. Каждая ячейка принадлежит одному складу.

## Stock Position
Текущее состояние конкретного запаса материала, определяемое сочетанием материала, склада, ячейки хранения, состояния запаса и партии (если используется).

## Stock State
Состояние складского запаса (AVAILABLE, RESERVED, IN_TRANSIT, BLOCKED и др.).

## Warehouse Operation
Внутренний объект Warehouse, описывающий изменение складских остатков.

## Warehouse Movement
Неизменяемая запись истории складского учета, создаваемая после успешного выполнения Warehouse Operation.

## Batch
Партия материала, используемая при партийном учете.

## Reservation
Резервирование материала путем изменения состояния Stock Position.

## Transfer
Документ или операция перемещения материалов между складами и (или) ячейками хранения.

## Receiving Cell
Ячейка приемки для временного размещения поступивших материалов.

## Inventory
Документ сверки фактических и учетных остатков.

## Adjustment
Документ исправления ошибок складского учета.

## Manual Issue
Документ ручного списания материалов.

## Split Transfer Document
Документ, созданный при разделении исходного документа перемещения на исполнимые и проблемные строки.

## Warehouse API
Публичный интерфейс Warehouse Capability.
