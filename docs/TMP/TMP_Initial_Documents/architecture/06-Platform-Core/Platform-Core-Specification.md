# Platform Core Specification

**Document ID:** TMP-SPEC-006

**Status:** Accepted

**Version:** 1.1

------------------------------------------------------------------------

# Назначение

Platform Core является инфраструктурным ядром TOP Manufacturing Platform
(TMP).

Platform Core предоставляет базовые механизмы работы платформы и не
содержит предметной бизнес-логики.

Platform Core обеспечивает единые инфраструктурные правила взаимодействия
между Capability, оставаясь независимым от их внутренней реализации.

------------------------------------------------------------------------

# Цели Platform Core

- запуск и остановка платформы;
- инициализация компонентов;
- управление жизненным циклом компонентов;
- предоставление общей инфраструктуры;
- обеспечение стабильной основы для развития платформы;
- обеспечение независимого развития Capability без изменения Platform Core.

------------------------------------------------------------------------

# Область ответственности

Platform Core отвечает исключительно за инфраструктуру платформы.

В его обязанности входит:

- запуск приложения;
- регистрация компонентов;
- управление жизненным циклом компонентов;
- предоставление инфраструктурных сервисов;
- конфигурация платформы;
- журналирование;
- инфраструктура аудита;
- инфраструктура событий;
- обработка критических ошибок.

Platform Core не отвечает за выполнение бизнес-операций Capability и не
управляет их внутренним состоянием.

------------------------------------------------------------------------

# Что НЕ входит в Platform Core

Platform Core не содержит:

- бизнес-логику;
- бизнес-объекты;
- документы;
- производственные процессы;
- складской учет;
- снабжение;
- аналитику;
- пользовательские сценарии;
- состояние функциональных областей;
- прямое изменение данных Capability.

------------------------------------------------------------------------

# Компоненты платформы

Platform Core знает только о существовании зарегистрированных
компонентов платформы.

Core не знает их назначения и внутренней реализации.

Каждый компонент реализует единый контракт Platform Component.

Platform Core не знает внутреннюю модель данных Capability и не имеет
прямого доступа к их бизнес-объектам.

------------------------------------------------------------------------

# Взаимодействие с компонентами

Platform Core:

- регистрирует компоненты;
- инициализирует компоненты;
- запускает компоненты;
- завершает работу компонентов;
- предоставляет инфраструктурные сервисы.

Platform Core не вызывает бизнес-функции компонентов напрямую.

Все взаимодействие выполняется через публичные контракты Capability.

Platform Core не обращается напрямую к внутренним данным Capability.

------------------------------------------------------------------------

# Инфраструктурные сервисы

Platform Core предоставляет инфраструктуру для:

- конфигурации;
- логирования;
- аудита;
- публикации событий;
- управления жизненным циклом приложения.

Конкретные сервисы реализуются отдельными компонентами платформы.

Platform Core предоставляет инфраструктуру, но не определяет бизнес-правила
использования этих сервисов.

------------------------------------------------------------------------

# Зависимости

Platform Core не имеет зависимостей от функциональных областей
платформы.

Все зависимости направлены от компонентов к Platform Core.

Platform Core не зависит от:

- Warehouse;
- Production;
- Procurement;
- Order Management;
- Security;
- других Capability.

------------------------------------------------------------------------

# Принципы проектирования

1. Независимость от предметной области.
2. Минимальная ответственность.
3. Расширяемость без изменения Core.
4. Отсутствие прямых зависимостей от компонентов.
5. Использование контрактов вместо знаний о реализации.
6. Platform Core не является владельцем бизнес-данных.
7. Platform Core не изменяет состояние Capability.
8. Capability взаимодействуют через публичные API.
9. Новые Capability подключаются без изменения Platform Core.

------------------------------------------------------------------------

# Architecture Rules

### AR-001

Platform Core никогда не содержит предметную бизнес-логику.

### AR-002

Platform Core не является владельцем бизнес-состояния платформы.

### AR-003

Каждая Capability самостоятельно управляет своим жизненным циклом после
регистрации.

### AR-004

Platform Core предоставляет инфраструктуру, но не управляет
бизнес-процессами.

### AR-005

Добавление новой Capability не должно требовать изменения Platform Core.
