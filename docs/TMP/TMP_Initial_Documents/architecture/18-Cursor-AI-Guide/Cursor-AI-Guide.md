# Cursor AI Guide

**Document ID:** TMP-DEV-002  
**Status:** Accepted  
**Version:** 1.0

---

# Назначение

Документ определяет обязательные правила работы Cursor AI при разработке TOP Manufacturing Platform (TMP).

Любое отклонение от данного документа считается нарушением процесса разработки.

---

# Основные принципы

1. Cursor не принимает архитектурные решения самостоятельно.
2. Cursor работает только в пределах поставленной задачи.
3. Перед написанием кода Cursor обязан выполнить Question Gate.
4. При недостатке информации Cursor обязан задать вопросы.
5. Cursor не выходит за границы текущего этапа разработки.
6. После завершения задачи Cursor обновляет документацию.

---

# Ограничение контекста

Cursor имеет право читать только:

- документы, указанные в задаче;
- текущий Implementation Plan;
- текущий Checklist;
- текущий Maven-модуль;
- связанные тесты.

Запрещено:

- анализировать весь репозиторий;
- самостоятельно искать информацию в других модулях;
- читать документы, не указанные в задаче.

---

# Ограничение размера задачи

Одна задача должна:

- реализовывать один пункт чек-листа;
- иметь один законченный результат;
- работать преимущественно в одном Maven-модуле;
- создавать не более 5 production-файлов;
- изменять не более 10 production-файлов.

Если задача превышает данные ограничения, Cursor обязан предложить разделить её.

---

# Question Gate

Перед началом реализации Cursor проверяет:

1. Достаточна ли спецификация.
2. Нет ли архитектурных противоречий.
3. Понятны ли критерии завершения.
4. Определены ли разрешённые файлы.
5. Не превышены ли лимиты задачи.
6. Не требуется ли изменение других модулей.
7. Не требуется ли новое архитектурное решение.
8. Не требуется ли уточнение пользователя.

Если хотя бы один ответ положительный:

- код писать запрещено;
- Cursor формулирует вопросы;
- ожидает ответа пользователя.

Если вопросов нет:

```text
QUESTION GATE: PASSED
```

только после этого начинается реализация.

---

# Правила реализации

Cursor обязан:

- соблюдать архитектуру проекта;
- соблюдать Development Guide;
- соблюдать Code Quality Standards;
- использовать только утверждённые технологии;
- реализовывать только текущую задачу.

Запрещено:

- менять архитектуру;
- изменять соседние модули;
- выполнять рефакторинг "заодно";
- обновлять зависимости без разрешения;
- исправлять несвязанный код;
- реализовывать будущую функциональность.

---

# Работа с JavaFX

Cursor обязан:

- использовать FXML;
- использовать CSS;
- использовать Navigation Service;
- использовать ViewModel;
- не внедрять Spring в Controller.

Запрещено:

- создавать бизнес-логику в Controller;
- обращаться к Repository из UI;
- напрямую загружать FXML вне Navigation Service.

---

# Документирование

После завершения задачи Cursor обязан:

- обновить Checklist;
- обновить Implementation Log;
- отметить выполненные пункты;
- сообщить результат пользователю.

Если задача не завершена, чек-лист обновлять запрещено.

---

# Проверки

Перед завершением задачи Cursor обязан выполнить:

- компиляцию проекта;
- необходимые тесты;
- архитектурные проверки;
- проверку критериев завершения.

При наличии ошибок задача считается незавершённой.

---

# Новый чат Cursor

Новое окно Cursor открывается:

- для новой задачи;
- для нового Maven-модуля;
- при смене этапа разработки;
- при приближении к пределу контекстного окна.

Продолжение работы в переполненном контексте запрещено.

---

# Запрещено

Cursor запрещено:

- принимать архитектурные решения;
- изменять спецификации;
- изменять утверждённые документы;
- добавлять новые зависимости без разрешения;
- читать лишние документы;
- писать код при наличии блокирующих вопросов;
- нарушать границы этапов разработки.

---

# Definition of Done

Задача считается завершённой только если:

- выполнены все пункты чек-листа;
- выполнены критерии приёмки;
- код успешно компилируется;
- проверки завершены успешно;
- документация обновлена;
- пользователь получил отчёт о выполнении.
