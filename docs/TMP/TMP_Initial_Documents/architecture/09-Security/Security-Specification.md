# Security Specification

**Document ID:** TMP-SPEC-009  
**Status:** Accepted  
**Version:** 2.0

---

# Назначение

Security — платформенный компонент TOP Manufacturing Platform (TMP), отвечающий за:

- пользователей;
- роли;
- права доступа;
- аутентификацию;
- авторизацию;
- пользовательские сессии;
- аудит операций безопасности.

Security не содержит предметной бизнес-логики и не зависит от функциональных модулей.

---

# Основные принципы

1. Все права объявляются Capability.
2. Security хранит пользователей, роли и права.
3. Security проверяет наличие прав, но не определяет их бизнес-смысл.
4. Проверка доступа обязательна для каждой бизнес-операции.
5. Интерфейс скрывает недоступные функции, но это не заменяет проверку доступа.
6. Все изменения прав и пользователей подлежат аудиту.

---

# Пользователь

Для пользователя хранятся:

- идентификатор;
- логин;
- имя;
- BCrypt-хеш пароля;
- назначенные роли;
- индивидуальные права;
- признак логического удаления.

Пользователь имеет два состояния:

- Активный;
- Удалённый (логически).

Физическое удаление пользователя запрещено.

---

# Пароль

Пароль:

- хранится только как BCrypt-хеш;
- никогда не сохраняется в открытом виде;
- никогда не записывается в журнал.

Пользователь может самостоятельно изменить пароль.

Администратор может выполнить сброс пароля.

---

# Первый администратор

Первый администратор создаётся автоматически при первом запуске системы.

Учётные данные задаются через конфигурацию и не хранятся в репозитории.

Повторное создание администратора не выполняется.

---

# Роль

Роль является шаблоном набора прав.

Пользователь может иметь несколько ролей.

Администратор может дополнительно назначать или отзывать индивидуальные права.

Индивидуальные права имеют приоритет над шаблоном роли.

---

# Право

Право — уникальное разрешение на выполнение операции.

Формат идентификатора:

```text
<area>.<resource>.<action>
```

Примеры:

```text
order.view
order.create
warehouse.issue
production.release
cutting.generate
```

После регистрации идентификатор права изменять запрещено.

---

# Capability

Каждая Capability самостоятельно объявляет:

- свои права;
- права для команд;
- права для экранов;
- права для документов;
- права для отчётов.

Security только регистрирует их.

---

# Проверка доступа

Проверка обязательна:

- при открытии экранов;
- при выполнении команд;
- при создании;
- при изменении;
- при удалении;
- при проведении документов;
- при выполнении любой бизнес-операции.

Окончательная проверка выполняется непосредственно перед выполнением операции.

---

# Рабочее место пользователя

После входа система формирует рабочее место на основании:

- зарегистрированных Capability;
- ролей пользователя;
- индивидуальных прав.

Недоступные разделы и команды пользователю не отображаются.

---

# Пользовательская сессия

Пользователь входит в систему при запуске приложения.

Сессия действует до:

- выхода пользователя;
- закрытия приложения.

Автоматическое завершение по времени не используется.

После выхода:

- сессия очищается;
- открытые экраны закрываются;
- отображается экран входа.

---

# Администрирование

Администратор может:

- создавать пользователей;
- удалять пользователей (логически);
- изменять пользователей;
- сбрасывать пароль;
- создавать роли;
- изменять роли;
- назначать роли;
- назначать индивидуальные права;
- просматривать аудит безопасности.

Самостоятельная регистрация пользователей отсутствует.

---

# Аудит

Фиксируются:

- вход;
- выход;
- создание пользователя;
- изменение пользователя;
- удаление пользователя;
- смена пароля;
- сброс пароля;
- изменение ролей;
- изменение индивидуальных прав.

Для каждого события сохраняются:

- дата и время;
- пользователь;
- операция;
- объект;
- краткое описание.

Пароли и их хеши никогда не журналируются.

---

# Ограничения

Запрещено:

- LDAP;
- Active Directory;
- OAuth;
- двухфакторная авторизация;
- восстановление пароля по электронной почте;
- внешние провайдеры авторизации;
- автоматическая блокировка пользователя;
- срок действия пароля;
- хранение пароля в открытом виде.

Роль нельзя удалить, пока она назначена хотя бы одному пользователю.

---

# Architecture Rules

**AR-001**  
Все права объявляются только Capability.

**AR-002**  
Security является единственным владельцем пользователей, ролей и прав.

**AR-003**  
Capability определяет необходимые права, Security проверяет их наличие.

**AR-004**  
Интерфейс не заменяет проверку доступа.

**AR-005**  
Каждая бизнес-операция обязана выполнять серверную проверку прав.
