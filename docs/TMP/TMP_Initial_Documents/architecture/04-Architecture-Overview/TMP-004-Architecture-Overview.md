# TMP-004 Architecture Overview

**Document ID:** TMP-004

**Status:** Accepted

**Version:** 1.0

------------------------------------------------------------------------

# Назначение

Документ описывает общую архитектуру TOP Manufacturing Platform (TMP),
основные компоненты платформы и связи между ними.

Подробная реализация каждого компонента приводится в соответствующих
спецификациях.

------------------------------------------------------------------------

# Общая архитектура

``` text
                    TOP Manufacturing Platform

                         Desktop Application
                                 │
                                 ▼
                          Platform Core
                                 │
        ┌────────────────────────┼────────────────────────┐
        ▼                        ▼                        ▼
 Platform Services         Functional Areas         Security
                                 │
        ┌────────────────────────┼────────────────────────┐
        ▼                        ▼                        ▼
 Order Management         Warehouse             Production
 Procurement              Analytics             Cutting
                                 │
                                 ▼
                          Documents
                                 │
                                 ▼
                         Business Objects
```

------------------------------------------------------------------------

# Архитектурные уровни

## Desktop Application

Единое десктопное приложение, предоставляющее пользователю доступ к
возможностям платформы.

------------------------------------------------------------------------

## Platform Core

Обеспечивает общие механизмы функционирования платформы и не содержит
предметной бизнес-логики.

------------------------------------------------------------------------

## Platform Services

Общие сервисы, используемые всеми функциональными областями платформы.

К ним относятся:

-   Document Engine
-   Capability Engine
-   Configuration
-   Audit
-   Search
-   Notification (в перспективе)

------------------------------------------------------------------------

## Functional Areas

Самостоятельные функциональные области платформы.

На текущем этапе:

-   Order Management
-   Warehouse
-   Production
-   Procurement
-   Cutting Optimization
-   Analytics

Каждая функциональная область отвечает только за собственную предметную
область.

------------------------------------------------------------------------

## Security

Подсистема управления пользователями, ролями и правами доступа.

------------------------------------------------------------------------

## Documents

Единый механизм изменения бизнес-состояния предприятия.

Все функциональные области взаимодействуют посредством документов.

------------------------------------------------------------------------

## Business Objects

Долговременные сущности предприятия, существующие независимо от
отдельных документов.

------------------------------------------------------------------------

# Основные зависимости

-   Desktop Application использует Platform Core.
-   Platform Core предоставляет сервисы платформы.
-   Functional Areas используют Platform Services.
-   Functional Areas не изменяют данные друг друга напрямую.
-   Все изменения выполняются через Documents.
-   Documents изменяют состояние Business Objects.

------------------------------------------------------------------------

# Принципы архитектуры

-   Единое приложение.
-   Документо-ориентированная архитектура.
-   Независимость функциональных областей.
-   Расширяемость без изменения Platform Core.
-   Инкрементальное развитие.
